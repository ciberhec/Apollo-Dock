# Notion Integration (Planned)

This document describes how Apollo Dock — specifically the **Domain Agent** tool — could pipe its output into a Notion database once the Notion API is wired up. This is a planning document; no code in this repo currently talks to Notion.

## Goal

Every time a support agent runs a domain analysis, the resulting report should be one click away from being saved as a new row in a Notion database. That database becomes a permanent record of every customer domain the team has triaged, queryable by status, provider, ticket ID, and remediation outcome.

## Proposed Notion database schema

| Property            | Type           | Notes                                                                                                  |
|---------------------|----------------|--------------------------------------------------------------------------------------------------------|
| Domain              | Title          | The customer domain analyzed.                                                                          |
| Ticket              | Rich text / URL| Optional Zendesk/Salesforce link, prompted from agent when saving.                                     |
| Overall Status      | Select         | `pass` / `warn` / `fail`.                                                                              |
| Provider            | Select         | `Google Workspace`, `Microsoft 365`, `Unknown`, …                                                      |
| SPF Status          | Select         | `pass` / `warn` / `fail` / `info`.                                                                     |
| DMARC Status        | Select         |                                                                                                        |
| DKIM Status         | Select         |                                                                                                        |
| Blacklist Status    | Select         |                                                                                                        |
| Raw SPF             | Rich text      |                                                                                                        |
| Raw DMARC           | Rich text      |                                                                                                        |
| DKIM Selector       | Rich text      |                                                                                                        |
| Blacklisted On      | Multi-select   | One tag per DNSBL the domain is listed on.                                                             |
| Customer Message    | Rich text      | The full customer-facing remediation message generated by the analyzer.                                |
| Agent               | Person         | Whoever ran the analysis.                                                                              |
| Run At              | Created time   | Auto-populated.                                                                                        |
| Re-check Due        | Date           | Optional — set 48 hours out so the agent can confirm DNS changes propagated.                           |

## How the wiring would look

1. Add `@notionhq/client` and a `NOTION_TOKEN` plus `NOTION_DOMAIN_DB_ID` to a local `.env`.
2. Create `src/tools/domain-agent/core/notion-integration.js` that exports `saveReport(report, { ticketUrl, agentEmail })`. It would:
   - Call `notion.pages.create({ parent: { database_id }, properties: { ... } })`.
   - Map each `report.findings[]` entry to the matching select / rich-text column above.
   - Return the new page URL so the renderer can show "Saved to Notion ↗".
3. Add a **Save to Notion** button next to **Copy to clipboard** in `index.html`. Disabled (with a tooltip) until `NOTION_TOKEN` is detected at app launch.
4. In `main.js`, register `ipcMain.handle('notion:save', ...)` that delegates to the integration module. Never expose the token to the renderer.

## Stretch ideas

- **Weekly digest:** a scheduled job that queries the database for `Overall Status = fail AND Re-check Due ≤ today` and posts the list to a Slack channel.
- **Linked customer page:** if the agent provides a customer account ID, the new Notion row can backlink to the customer's CRM page via Notion relations.
- **Automatic ticket comment:** after saving, optionally post the same remediation message as a private internal note on the linked Zendesk ticket.

## Security notes

- The Notion integration token is workspace-scoped — restrict it to the single domain-triage database, not the whole workspace.
- Never bundle the token. Read it from `.env` at runtime; document this in the README's API keys section.
- Anything sent to Notion is internal company data; the integration must not ingest customer secrets (the analyzer only handles public DNS records, so this is already safe by construction).
